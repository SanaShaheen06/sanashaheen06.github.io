The Goal

Our primary objective was to deploy an enterprise-grade messaging platform that was both resilient and secure. This meant it couldn't have single points of failure. The system required a redundant database, multiple application servers, and centralized user management that connected to our company's Active Directory.

My Solution & Process

I began by configuring two Windows Server 2019 machines as Domain Controllers to handle LDAP authentication and DNS. For database redundancy, I deployed a three-server MongoDB replica set on Ubuntu, ensuring that if one database server failed, the others would take over instantly. Next, I set up two separate Ubuntu servers for the Rocket.Chat application itself, using PM2 to manage the Node.js processes and ensure they were always running.

To secure the platform and distribute user traffic, I installed and configured NGINX as a reverse proxy. This handled all incoming HTTPS traffic, encrypting communications, and load-balancing requests between the two Rocket.Chat servers. Finally, I integrated Rocket.Chat with Active Directory using LDAP, which allowed users to log in with their existing company credentials.
