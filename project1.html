Academic Capstone Project

High-Availability Hybrid IT Infrastructure Deployment

This project involved designing and deploying a secure, high-availability hybrid IT infrastructure for a simulated multinational organization with two sites â€” New York (Headquarters) and Hong Kong (Branch Office). The goal was to create a scalable and fault-tolerant environment supporting collaboration, helpdesk, and monitoring services across both locations.

Key Objectives

  • Build a redundant Active Directory environment for centralized user management.
  • Deploy Linux and Windows services with high availability and failover capabilities.
  • Enable secure communication, authentication, and cross-domain trust between both sites.
  • Implement monitoring, role-based access, and documentation for operational efficiency.

My Role & Responsibilities

I managed the Hong Kong domain (Capstone.hk) and handled key server and network configurations:

  • Deployed Primary and Secondary Domain Controllers (DC1, DC2) for redundancy and load balancing.
  • Installed and configured a Linux-based DHCP server for automatic IP management.
  • Established a two-way trust with the New York domain for secure resource sharing.
  • Managed users, groups, and Organizational Units (OUs) within Active Directory.
  • Applied Group Policy Objects (GPOs) for security, access control, and standardization.
  • Integrated Active Directory authentication with Mattermost and Zammad (SSO support).
  • Deployed Nagios for monitoring uptime, server health, and inter-site connectivity.
  • Created detailed documentation including network diagrams, configurations, and recovery steps.

Technologies Used

Windows Server 2019Ubuntu Linux Active DirectoryDNS & DHCP ZammadNagios MattermostPostgreSQL Group PolicyADCS (Planned) VMware vSphereVirtual Networking

Project Execution

The project simulated a global enterprise setup. I deployed and configured the Hong Kong domain to communicate securely with the New York site through trust relationships. Services like DHCP, DNS, and GPOs were configured to automate and secure network management. Linux-based applications, including Zammad (ticketing system) and Nagios (monitoring tool), were integrated with the Windows infrastructure, demonstrating seamless cross-platform compatibility.

For collaboration, Mattermost was implemented and connected with Active Directory for centralized login management. VMware vSphere provided virtualization for testing, scalability, and fault tolerance across servers. To strengthen security, planning was completed for deploying Active Directory Certificate Services (ADCS) for certificate-based authentication.

Outcomes & Impact

  • Successfully built a functioning hybrid IT infrastructure connecting two geographic sites.
  • Established centralized authentication with domain trust and redundancy.
  • Integrated Linux and Windows systems securely within one unified environment.
  • Implemented continuous system monitoring and proactive alerting using Nagios.
  • Produced clear, professional technical documentation and topology reports.

Key Learnings

This project enhanced my expertise in network configuration, Windows and Linux server management, and multi-site system design. It improved my ability to troubleshoot cross-platform authentication, enforce security policies, and document enterprise-grade infrastructure deployments.
