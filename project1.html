Academic Capstone Project

High-Availability Hybrid Infrastructure

This capstone project focused on designing and deploying a secure, high-availability hybrid IT infrastructure for a simulated multinational organization with two major sites. New York (Headquarters) and Hong Kong (Branch Office). The objective was to build a reliable and scalable environment supporting collaboration, helpdesk, and monitoring services across both locations.

My role focused on the Hong Kong domain (Capstone.hk), where I:

  • Domain Setup: Configured Primary Domain Controller (DC1) and Secondary Domain Controller (DC2) for redundancy and load balancing.
  • DHCP Deployment: Installed and configured Linux-based DHCP server to assign and manage IP addresses for client machines dynamically.
  • Trust Relationship: Established two-way trust between the Hong Kong and New York domains, enabling cross-domain authentication and resource sharing.
  • User and Group Management: Created and managed user accounts, security groups, and organizational units (OUs) through Active Directory
  • GPO Implementation: Applied Group Policy Objects to enforce security settings, login restrictions, and resource access policies.
  • Application Integration: Assisted in integrating Active Directory authentication with Mattermost and Zammad to enable single sign-on (SSO)
  • Documentation: Created technical documentation outlining network topology, server configurations, and recovery procedures.

The Goal

The core objective was to build a realistic, multi-site IT environment from scratch. This required creating two distinct but interconnected domains (New York and Hong Kong) to simulate a global enterprise, ensuring centralized user management and secure integration of both Windows and Linux applications.

My Solution & Process

My work centered on the Hong Kong domain. I began by deploying Active Directory and DHCP on Windows Server, establishing the foundation for all user and device management. Using Group Policy Objects (GPOs), I enforced security settings and standardized user environments. A critical milestone was configuring a domain trust with the New York site, which enabled seamless cross-domain resource access.

A key feature of this project was cross-platform integration. I successfully integrated several Linux-based services into our Windows AD environment, including a Zammad ticketing system and a Nagios monitoring server, proving that open-source tools could be managed within a Microsoft-centric infrastructure.

Key Takeaways

This project was invaluable for strengthening my skills in infrastructure deployment, cross-platform integration, and system hardening. I learned to troubleshoot complex authentication issues between operating systems and the critical importance of meticulous planning in a multi-site network.
