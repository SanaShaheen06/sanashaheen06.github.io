High-Availability Hybrid Infrastructure

This capstone project focused on designing and deploying a secure, high-availability hybrid IT infrastructure for a simulated multinational organization with two major sites â€” New York (Headquarters) and Hong Kong (Branch Office). The objective was to build a reliable and scalable environment supporting collaboration, helpdesk, and monitoring services across both locations.

Key Goals

  • Establish a redundant Active Directory environment for centralized user management.
  • Deploy high-availability Linux and Windows services across two sites.
  • Implement secure communication, user authentication, and role-based access between locations.
  • Ensure service continuity and fault tolerance through redundancy and replication mechanisms.

My Role & Responsibilities

My role focused on the Hong Kong domain (Capstone.hk), where I:

  • Configured Primary Domain Controller (DC1) and Secondary Domain Controller (DC2) for redundancy and load balancing.
  • Installed and configured Linux-based DHCP server to assign and manage IP addresses dynamically.
  • Established two-way trust between Hong Kong and New York domains for cross-domain authentication.
  • Created and managed user accounts, security groups, and organizational units (OUs) through Active Directory.
  • Applied Group Policy Objects (GPOs) to enforce security settings and resource access controls.
  • Assisted in integrating Active Directory authentication with Mattermost and Zammad for single sign-on (SSO).
  • Deployed Nagios for server and network monitoring between sites.
  • Created detailed documentation of network topology, configurations, and recovery procedures.

The Goal

The core objective was to build a realistic, multi-site IT environment from scratch. This required creating two distinct but interconnected domains (New York and Hong Kong) to simulate a global enterprise, ensuring centralized user management and secure integration of both Windows and Linux applications.

My Solution & Process

My work centered on the Hong Kong domain. I began by deploying Active Directory and DHCP on Windows Server, establishing the foundation for user and device management. Using Group Policy Objects (GPOs), I enforced security settings and standardized user environments. A critical milestone was configuring a domain trust with the New York site, enabling seamless cross-domain resource access.

A key feature of this project was cross-platform integration. I successfully integrated several Linux-based services into the Windows AD environment, including a Zammad ticketing system and a Nagios monitoring server, demonstrating that open-source tools could function securely within a Microsoft-centric infrastructure.

Detailed Overview

The project involved designing a high-availability infrastructure leveraging both Windows and Linux systems. Redundant domain controllers were deployed for load balancing, while DHCP and DNS services ensured consistent IP and name resolution across both locations. Group policies were implemented for security baselines, login control, and system updates.

For collaboration and IT service management, Mattermost was deployed as an enterprise messaging platform and Zammad as the helpdesk solution, both backed by PostgreSQL databases. Monitoring was handled via Nagios, which provided real-time alerts on server health, disk usage, and network latency.

To enhance security, planning for Active Directory Certificate Services (ADCS) was included for certificate-based authentication and encryption. The entire setup ran on VMware vSphere, allowing for isolated testing, scalability, and resource management.

Outcomes & Results

  • Successfully deployed a fully functional hybrid IT environment connecting New York and Hong Kong domains.
  • Implemented domain trust and centralized authentication across sites.
  • Achieved redundancy through multi-server architecture and DNS replication.
  • Integrated open-source and Microsoft services securely and efficiently.
  • Produced comprehensive documentation and topology diagrams for assessment.

Key Takeaways

This project strengthened my skills in infrastructure deployment, cross-platform integration, and troubleshooting complex authentication issues between systems. It emphasized the value of meticulous planning, redundancy, and monitoring in maintaining a resilient multi-site network.
